# 购车决策分析方法论

> 从本次七车调研（SU7、YU7、Model Y、ET5、ES6、007GT、001）中提炼的完整流程
> 可复用为通用的汽车购买深度调研 skill

---

## 总览

```
Phase 0: 建立决策框架
    ↓
Phase 1: 候选车基础档案（每车独立）
    ↓
Phase 2: 负面信息专项（每车/每品牌独立）
    ↓
Phase 3: 安全事故率标准化分析（跨车对比）
    ↓
Phase 4: 行业销量与数据置信度（对比基线）
    ↓
Phase 5: 横向对比表（按框架分层）
    ↓
Phase 6: 最终决策矩阵（加权评分 + 敏感性分析）
```

每个 Phase 的产出是独立文件，后一个 Phase 依赖前一个的数据。

## Phase 0: 建立决策框架

### 目的
在看任何车之前，先定义"好车"的标准。防止被营销、外观、品牌带偏。

### 产出
`first-principles.md` — 选车决策分层模型

### 方法
从第一性原理出发：**车的本质是将人安全地从 A 带到 B。**

围绕这个定义，按"离本质的距离"分成五层，层级越低越重要：

| 层级 | 问题 | 权重（建议） |
|------|------|-------------|
| 1. 核心层 | 能不能安全到达？ | 35% |
| 2. 效率层 | 能不能高效到达？ | 25% |
| 3. 成本层 | 付出多大代价？ | 20% |
| 4. 体验层 | 过程舒不舒服？ | 15% |
| 5. 符号层 | 这车在替你说什么？ | 5% |

### 关键原则
- **决策顺序必须自上而下**：核心层不过关，不往下看
- 符号层权重最低，但大多数人实际决策时权重最高——这是最常见的错误
- 权重可根据个人偏好调整，但调整本身应该是有意识的，不是被营销驱动的

## Phase 1: 候选车基础档案

### 目的
为每款候选车建立完整、结构化的事实档案。

### 产出
每车一个文件：`{brand}-{model}.md`（如 `xiaomi-su7.md`、`nio-et5.md`）

### 调研维度（沿用框架五层）

| 维度 | 对应层级 | 搜索要点 |
|------|---------|---------|
| 价格与版本 | 成本层 | 全系售价、各版本配置差异、选装包、限时政策 |
| 三电系统 | 核心层+效率层 | 电池规格、CLTC/WLTP续航、充电速度、电压平台、动力参数 |
| 底盘与安全 | 核心层 | 碰撞测试成绩（C-IASI/Euro NCAP/IIHS）、车身结构、气囊数量、主被动安全配置 |
| 智驾系统 | 效率层 | 芯片方案、算力、传感器配置、功能层级（基础/高速/城市）、实际表现 |
| 空间与实用性 | 效率层 | 车身尺寸、后排空间、后备箱容积、座椅放倒比例 |
| 补能体系 | 核心层+效率层 | 充电网络覆盖、快充速度、特殊补能方式（如换电） |
| 保养与售后 | 成本层 | 保养周期与费用、服务网络、二手残值/保值率 |
| 车机与生态 | 体验层 | 座舱芯片、屏幕、语音交互、OTA频率、生态兼容性 |

### 数据规范
- 每个数据点标注来源和年份
- 区分"官方数据" vs "媒体实测" vs "用户反馈"
- 如果同一指标有多个来源且数据冲突，全部列出并标注

### 搜索策略
每车执行 8-10 次 WebSearch，覆盖：
- 官方配置表 / 官网
- 权威媒体评测（新出行、42号车库、汽车之家、懂车帝）
- 国际媒体（Autocar、Car and Driver）用于补充海外测试数据
- 用户论坛 / 知乎长文（真实使用反馈）

### 特殊处理
- **有独特商业模式的车型**（如蔚来BaaS车电分离），在价格和补能维度增加专项分析
- **换代车型**，需明确标注"哪一代"的数据，避免混淆
- **同平台多车型**（如ET5/ET5T），在同一档案中对比差异

## Phase 2: 负面信息专项

### 目的
系统性收集每款车/品牌的问题、风险、投诉，**只记录负面，不做正面平衡**。与基础档案分开，避免"优缺点各半"的平衡谬误稀释风险信号。

### 产出
每车/每品牌一个文件：`negatives-{brand/model}.md`

如果同一品牌多款车共享品牌级风险（如蔚来的财务风险），合并为品牌级负面档案。

### 搜索矩阵

| 方向 | 搜索词模板 |
|------|-----------|
| 安全事故 | "{品牌} {车型} 事故 起火 自燃 碰撞 死亡 {年份}" |
| 品控投诉 | "{品牌} {车型} 质量问题 投诉 异响 故障 车质网 黑猫投诉" |
| 召回记录 | "{品牌} 召回 市场监管总局 {年份}" |
| 智驾事故 | "{品牌} 辅助驾驶 NOA NOP Autopilot 事故 失灵" |
| 售后维权 | "{品牌} 售后 维权 退车 拒保 服务差" |
| 续航虚标 | "{品牌} 续航虚标 冬季衰减 实际续航" |
| 财务风险 | "{品牌} 亏损 裁员 资金链 {年份}"（适用于新势力） |
| 降价维权 | "{品牌} 降价 老车主 维权 补偿" |

### 分类与排序
按严重程度从高到低排列：
1. **致死事故** — 最高优先级
2. **安全召回** — 系统性缺陷
3. **品控批量问题** — 影响日常使用
4. **财务/经营风险** — 影响品牌存续（仅新势力）
5. **服务与售后问题** — 影响持有体验
6. **其他**

### 验证状态标注
每条负面信息标注：
- `已证实（多源验证）` — 多家权威媒体报道
- `已证实（单源）` — 仅一家媒体但有实证
- `待验证` — 用户爆料、论坛信息
- `厂商已回应` — 标注回应内容

## Phase 3: 安全事故率标准化分析

### 目的
将事故绝对数标准化为"每万辆"比率，消除"卖得多所以事故多"的分母偏差。

### 产出
`safety-rate-analysis.md` + `research-safety-rate.md`（方法论）

### 核心公式

```
每万辆事故率 = 已知事故数 ÷ 累计交付量 × 10,000
```

### 数据矩阵

| 指标 | 分子（事故数） | 分母（保有量） | 来源 |
|------|--------------|--------------|------|
| 起火率 | 已知起火事件数 | 累计交付量 | 媒体报道 + 消防数据 |
| 致死率 | 已知致死事故数 | 累计交付量 | 媒体报道 |
| 召回率 | 召回总数量 | 累计交付量 | 市场监管总局公告 |
| 智驾事故率 | ADAS介入时事故数 | 智驾总里程 | NHTSA（仅Tesla可得） |
| 逃生失败率 | 碰撞后车门无法打开案例 | 碰撞总数 | 媒体报道（数据极不完整） |

### 对比基线
- 电动车行业平均起火率：~0.01-0.03%
- 燃油车火灾率：~0.01-0.02%（作为锚点）
- Tesla全球安全报告（每百万英里事故率）

### 关键方法论问题

#### 1. 数据置信度矩阵
不同品牌的数据可信度不同，必须显式标注：

| 品牌 | 数据来源 | 置信度 | 原因 |
|------|---------|--------|------|
| Tesla | NHTSA强制报告 + 官方季度报告 | **高** | 法定义务，最接近全量数据 |
| 蔚来/小米 | 媒体报道拼凑 | **中低** | 中国无强制公开事故数据库 |
| 比亚迪等 | 媒体报道 | **低** | 话题性低，报道不充分 |

#### 2. 报道偏差校正
Tesla和小米因话题性高，同样事故被报道的概率远高于比亚迪/传统品牌。这导致：
- 它们的"已知事故数"被系统性高估
- 不能直接用媒体报道数做跨品牌对比
- 只能说"已知事故率的上限"

#### 3. 保有量计算
- 优先用官方财报交付数据（最准确）
- 累计保有量 ≈ 逐年交付量求和（忽略报废，新能源车报废率极低）
- 注意区分"全球"vs"中国"保有量

### 额外维度
- **保险定价系数**：保险公司的风险定价是最真实的安全评估（但数据通常不公开）
- **碰撞后逃生风险**：电子门把手断电失效是跨品牌的结构性缺陷
- **品牌财务可持续性**：仅对新势力适用，但直接影响售后和零部件供应

## Phase 4: 行业销量与数据置信度

### 目的
建立行业大盘，为 Phase 3 的事故率数据提供分母校准和置信度判断。

### 产出
`industry-sales-context.md`

### 内容
1. **各品牌年度销量表**（2024-2025）
2. **20万+细分市场 Top 10**（与候选车直接竞争的价格带）
3. **行业整体规模**（新能源渗透率、保有量总数）
4. **保有量 → 事故数据可信度映射**：保有量越大，媒体报道事故占真实事故的比例越低
5. **报道偏差分析**：解释为什么不同品牌的"已知事故数"不能直接对比

### 关键判断
- 保有量 < 10万辆的车型，事故率统计学意义弱
- 保有量 > 100万辆的品牌，媒体报道仅覆盖真实事故的冰山一角
- 低话题度品牌（如比亚迪主力低端车型）的事故数据严重低估

## Phase 5: 横向对比表

### 目的
将各车档案的数据拉平到同一张表中，按框架五层逐项对比。

### 产出
`comparison-v2.md` — 按第一性原理框架组织的全维度对比

### 结构

每一层包含：
1. **事实表格**：纯数据，不加判断
2. **事实判断**：基于数据的客观分析
3. **层级结论**：该层的排名和关键区分点

### 各层具体对比项

**核心层**：
- 主被动安全（碰撞认证、气囊、车身强度）
- 机械可靠性（上市时长、召回、投诉、品控问题、额外风险）
- 续航与补能（入门续航、最长续航、冬季打折率、抛锚风险）

**效率层**：
- 补能效率（电压平台、快充速度、充电网络、换电）
- 智驾减负（城市NOA覆盖、实际体验、硬件配置、共同缺陷）
- 空间利用率（车型、后备箱、后排放倒、后排乘坐）
- 停车与城市通过性（车宽/长/高、地库友好度）

**成本层**：
- 购车成本（入门价、补齐智驾真实价、空悬+长续航价）
- 持有成本（保养、保险、电池租金、电费、年均合计）
- 折旧与残值（1年/3年保值率、核心风险）

**体验层**：
- 悬架、座椅、NVH、车机、加速、驾驶乐趣、独特体验

**符号层**：
- 品牌认知、社会信号、设计语言、争议性

### 附加内容
- **决策路径流程图**：按 Q1(核心层否决)→Q2(效率层场景)→Q3(成本层预算)→Q4(体验+符号偏好) 的顺序引导决策
- **仍需复核的关键事实**：标注数据中尚未验证或可能过时的条目

## Phase 6: 最终决策矩阵

### 目的
将定性对比转化为定量评分，输出可操作的推荐排名。

### 产出
`decision-matrix.md`

### 方法

#### 1. 评分（10分制）
- 每层拆分为 3-4 个子维度，逐项打分
- 每个分数附带扣分/加分理由
- 未经验证的宣称不给加分
- 存在结构性风险的酌情扣分

#### 2. 加权总分
```
总分 = Σ(各层得分 × 各层权重)
```

#### 3. 敏感性分析
改变权重，观察排名是否翻转：

| 场景 | 权重调整 | 回答的问题 |
|------|---------|-----------|
| 安全第一型 | 核心层 50% | 最看重验证过的安全 |
| 家庭实用型 | 效率层 40% | 最看重空间+智驾+补能 |
| 预算敏感型 | 成本层 40% | 最看重性价比 |
| 风险消除型 | 移除特定风险扣分 | 如果某品牌风险消失会怎样 |

敏感性分析的价值：**如果换一种合理权重排名就翻转，说明差距不大，应该靠场景匹配而非分数选车。**

#### 4. 一票否决清单
列出硬性排除条件，帮助快速缩小候选范围：
- 无法接受的风险（如品牌停运、无碰撞数据）
- 硬性需求不满足（如必须SUV、预算硬顶）
- 使用场景不兼容（如无家充+无换电站）

#### 5. 买家画像推荐
按典型用户画像给出推荐，而非只给一个排名：
- 预算有限要全配 → ?
- 家庭用车要空间 → ?
- 极度厌恶风险 → ?
- 追求驾控 → ?

#### 6. 终极决策树
二叉树结构，每个节点是一个是/否问题，叶子节点是推荐车型。

## 转为 Skill 时的设计要点

### 输入
- 候选车型列表（2-6款）
- 用户预算范围
- 用户使用场景（可选：有无家充、通勤距离、是否需要SUV、是否长途多）
- 用户风险偏好（可选：保守/中性/激进）

### 执行流程
1. **Phase 0** 自动加载（框架固定，不需要每次重建）
2. **Phase 1-2** 并行执行（每车一个 Agent，基础档案 + 负面信息可合并或拆分）
3. **Phase 3** 在 Phase 1-2 完成后执行（需要交付量数据做分母）
4. **Phase 4** 可与 Phase 3 并行（行业数据独立于候选车数据）
5. **Phase 5** 在 Phase 1-4 全部完成后执行（汇总）
6. **Phase 6** 在 Phase 5 完成后执行（评分）

### 可配置参数
- 五层权重（默认 35/25/20/15/5，可按用户偏好调整）
- 是否包含敏感性分析（默认是）
- 是否生成决策树（默认是）
- 调研深度：快速（Phase 1+5+6） vs 标准（Phase 0-6） vs 深度（含 Phase 3-4）

### 质量保证
- 每个数据点必须有来源
- 区分"事实"和"判断"
- 负面信息与正面信息物理隔离（不同文件）
- 评分理由可追溯
- 敏感性分析防止单一权重下的误导

## 本次调研的关键教训

1. **负面信息必须单独收集**。混在基础档案里会被正面信息稀释，失去预警作用。
2. **绝对事故数毫无意义**。不标准化到保有量，就会得出"卖得多=更危险"的错误结论。
3. **数据置信度比数据本身更重要**。中国没有NHTSA级别的公开数据库，所有基于媒体报道的事故率都是下限估计，且存在系统性报道偏差。
4. **品牌财务风险是隐藏的核心层问题**。对于新势力品牌，"公司还在不在"比"车好不好"更基本。
5. **BaaS等新商业模式需要单独建模**。不能简单比裸车价，要算总拥有成本（TCO）。
6. **所有ADAS都有相同的致命缺陷**。无法可靠识别静止障碍物——这不是某一家的问题，是当前技术的边界。
7. **敏感性分析是最诚实的工具**。如果换一种合理权重排名就变了，说明差距不大，应该靠场景匹配选车，而非迷信总分。
